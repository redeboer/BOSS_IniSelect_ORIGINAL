NTuples (eventual TTrees and their branches). NTuples are like vectors, but its members do not necessarily have to be of the same type. In this package, the NTuples are used to store event-by-event information. Its values are then written to the output ROOT file, creating a ROOT TTree. In that sense, each NTuple here represents one TTree within that output ROOT file, and each NTuple::Item represents its leaves. The name of the leaves is determined when calling NTuple::addItem. Note that the NTuple::Items are added to the NTuple during the PipiJpsi::initialize() step. This is also the place where you name these variables, so make sure that the structure here is reflected there!

Modified header of the stock BOSS package PipiJpsi (version 00-00-03).

Analyse and events. This example package particularly teaches the use of Monte Carlo truth.

Constructor for the PipiJpsi algorithm.

Here, you should declare properties: give them a name, assign a parameter (data member of RhopiAlg), and if required a documentation string. Note that you should define the paramters themselves in the header (RhopiAlg/RhopiAlg.h) and that you should assign the values in share/jopOptions_RhopiAlg.txt. Algorithms should inherit from Gaudi's Algorithm class. See https://dayabay.bnl.gov/dox/GaudiKernel/html/classAlgorithm.html.

◆ execute()

StatusCode PipiJpsi::execute ( )

Inherited execute method of the Algorithm. This function is called for each event.

  1. Log stream and status code

  2. Event counter and set new file

  3. Print trigger information once

  4. Asign four-momentum with KalmanTrack

    • Declare vectors etc.

    • Create selection of 'good' charged tracks

      • Get MDC Kalman track for this charged track

      • Get , geometry

      • Apply vertex cuts

      • Store this track as 'good'

      • Count if either pion or lepton. The track is considered to be pion if the probability to be an electron is less than 1.

      • Store 4-momenta of the four different cases (pion or lepton and negative or positive)

    • Test selections (apply cuts)

      • Apply photon cut: only 3 or 4 photons

      • Compute total energy

      • Store vectors in either muon or electron case

      • Apply cuts

  5. With momentum method calculate the invariant mass of Jpsi actually we use the recoil mass

  6. mass cut

  7. Angle difference between pions, suppress gamma convertion.

    Attention
    This is an important part of the PipiJpsi algorithm!

    • Compute angle between pions

    • Magnitude of the 3-momenta

    • Compute ratios

    • Apply boost

    • Compute total masses, energies, and momenta

    • Store indices

    • Store momenta in matrix when one track is assumed missing

    • Set filter passed

  8. Monte Carlo information.
    Attention
    This is an important part of the PipiJpsi algorithm!

  9. Selection of 'good' photons. Note that these aren't really used, as the process doesn't involve photons.

  10. Finish Good Photon Selection

  11. Check dedx infomation

  12. Check TOF infomation
