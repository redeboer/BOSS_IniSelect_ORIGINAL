Analyse events.

This package has been created as an extension of the D0phi_KpiKK package. The motivation for reconstructing the meson through the decay channel (extra ) is that (1) its branching ratio is much larger ( versus for ) and (2) that the extra excludes the that we see as a large backround in the D0phi_KpiKK package (due to parity). Specification of TrackSelector::CreateMCTruthSelection.

Create selection of MC truth particles by looping over the collection of MC particles created by TrackSelector::CreateMCTruthCollection().

  1. Abort if input file is not from a Monte Carlo simulation (that is, if the run number is not negative).
  2. Abort if "write_fit_mc", has been set to false.
  3. Clear MC truth particle selections.
  4. Loop over fMcParticles collection of MC truth particles and fill the selections.

StatusCode D0phi_KpipiKK::execute_rest ( )
virtual

Inherited execute method of the Algorithm.

This function is called for each event.

  1. Charged track cut: Apply a strict cut on the number of particles. Only 4 charged tracks in total.

  2. Net charge cut: Apply a strict cut on the total charge detected in the detectors. If this charge is not , this means some charged tracks have not been detected.

  3. Create selection of charged tracks.

    1. Initialise PID and skip if it fails:

      • use probability method

      • use and the three ToF detectors. Since data reconstructed with BOSS 7.0.4, ParticleID::useTofCorr() should be used for ToF instead of e.g. useTof1.

      • identify only pions and kaons

      • use

    2. Write Particle Identification information of all tracks

    3. Identify type of charged particle and add to related vector: (this package: kaon and pion).

      The particle identification first decides whether the track is more likely to have come from a pion or from a kaon.

      A cut is then applied on whether the probability to be a pion (or kaon) is at least fCut_PIDProb_min (see eventual settings in D0phi_KpipiKK.txt).

      Finally, the particle ID of the RecMdcKalTrack object is set to pion

      and the (positive) pion is added to the vector of pions.

  4. Create selection neutral tracks (photons)

    1. Get EM calorimeter info.

    2. Find angle differences with nearest charged pion.

      • Get the extension object from MDC to EMC.
      • Get angles in radians.

    3. Convert angles from radians to degrees.

    4. Write photon info ("photon" branch).

    5. Apply angle cut on the photons: you do not want to photons to be too close to any charged track to avoid noise from EMC showers that came from a charged track.

    6. Add photon track to vector for gammas

  5. Write the multiplicities of the selected particles.

  6. PID cut: apply a strict cut on the number of the selected particles. Only continue if:

    1. 2 negative kaons

    2. 1 positive kaon

    3. 1 positive pion

    4. at least 2 photons ( 's)

  7. Write PID information ("dedx_*" branchs)

  8. Loop over MC truth of final decay products.

    1. Only continue if the two kaons are different.

    2. Check topology: only consider that combination which comes from .

    3. Write 'fake' fit results, that is, momenta of the particles reconstructed from MC truth.

  9. Perform Kalman 5-constraint Kalman kinematic fit for all combinations and decide the combinations that results in the 'best' result. The 'best' result is defined as the combination that has the smallest value of: (that is the combination for which the invariant mass of the is closest to ). See D0phi_KpipiKK::MeasureForBestFit for the definition of this measure.

    Todo:

    1. Reset best fit parameters (see KKFitResult_D0phi_KpipiKK).

    2. Loop over all combinations of , , , and .

      1. Only continue if we are not dealing with the same kaon and/or photon.

      2. Get Kalman tracks reconstructed by the MDC.

      3. Get W-tracks.

      4. Initiate vertex fit.

      5. Test vertex fit.

      6. Get Kalman kinematic fit for this combination and store if better than previous one.

        Remarks
        5th constraint: resonance

        1. Apply max. cut (determined by fCut_PIDChiSq_max).

        2. Construct fit result object for this combintation.

        3. Write results of the Kalman kinematic fit (all combinations, "fit5c_all").

        4. Decide if this fit is better than the previous.

    3. Write results of the Kalman kitematic fit of the best combination ("fit5c_best" branches).

    4. If there is a fit result, write the MC truth topology for this event. Also increment fCutFlow_NFitOK counter if fit worked.

(Inherited) initialize step of Algorithm.

This function is called only once, when the algorithm is initialised.

Remarks
Define and load NTuples here. Other NTuples have already been defined in the TrackSelector::initilize step prior to this this method.

  1. "mult_select": Multiplicities of selected particles

    1. "NPhotons": Number of .

    2. "NKaonPos": Number of .

    3. "NKaonNeg": Number of .

    4. "NPionPos": Number of .

  2. "dedx_K" and "dedx_pi": energy loss PID branch. See TrackSelector::AddNTupleItems_Dedx for more info.

  3. "fit5c_*": results of the Kalman kinematic fit results. See TrackSelector::AddNTupleItems_Fit for more info.

  4. "photon": information of the selected photons

    1. "E": Energy of the photon.

    2. "px": component of the 5-momentum of the photon (computed from the detected angles).

    3. "py": component of the 5-momentum of the photon (computed from the detected angles).

    4. "pz": component of the 5-momentum of the photon (computed from the detected angles).

    5. "phi": Smallest angle between the photon and the nearest charged pion.

    6. "theta": Smallest angle between the photon and the nearest charged pion.

    7. "angle": Smallest angle between the photon and the nearest charged pion.

  5. "topology": Add extra mass branches to the MC truth branch for the topoana package

    1. "chi2": of the Kinematic kalman fit.

    2. "mpi0": Invariant mass of the candidate in .

    3. "mD0": Invariant mass of the candidate in the .
