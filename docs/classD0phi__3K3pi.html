Specification of TrackSelector::CreateMCTruthSelection.

Create selection of MC truth particles by looping over the collection of MC particles created by TrackSelector::CreateMCTruthCollection().

  1. Abort if input file is not from a Monte Carlo simulation (that is, if the run number is not negative).
  2. Abort if "write_fit_mc", has been set to false.
  3. Clear MC truth particle selections.
  4. Loop over fMcParticles collection of MC truth particles and fill the selections.

◆ execute_rest()

StatusCode D0phi_3K3pi::execute_rest ( )
virtual

Inherited execute method of the Algorithm.

This function is called for each event.

  1. Charged track cut: Apply a strict cut on the number of particles. Only 4 charged tracks in total.

  2. Create specific charged track selections

    1. Initialise PID and skip if it fails:

      • use probability method

      • use and the three ToF detectors. Since data reconstructed with BOSS 7.0.4, ParticleID::useTofCorr() should be used for ToF instead of e.g. useTof1.

      • identify only pions and kaons

      • use

    2. Write Particle Identification information of all tracks

    3. Identify type of charged particle and add to related vector: (this package: kaon and pion).

      The particle identification first decides whether the track is more likely to have come from a pion or from a kaon.

      A cut is then applied on whether the probability to be a pion (or kaon) is at least fCut_PIDProb_min (see eventual settings in D0phi_3K3pi.txt).

      Finally, the particle ID of the RecMdcKalTrack object is set to pion

      and the (positive) pion is added to the vector of pions.

  3. Write the multiplicities of the selected particles.

  4. PID cut: apply a strict cut on the number of the selected particles. Only:

    1. 2 negative kaons

    2. 1 positive kaon

    3. 1 positive pion

  5. Write PID information ("dedx_*" branchs)

  6. Loop over MC truth of final decay products.

    1. Only continue if the two kaons are different.

    2. Check topology: only consider that combination which comes from .

    3. Write 'fake' fit results, that is, momenta of the particles reconstructed from MC truth.

  7. Perform Kalman 4-constraint Kalman kinematic fit for all combinations and decide the combinations that results in the 'best' result. The 'best' result is defined as the combination that has the smallest value of: (that is the combination for which the invariant mass of the is closest to ). See D0phi_3K3pi::MeasureForBestFit for the definition of this measure.

    1. Construct fit result object for this combintation.

    2. Write results of the Kalman kinematic fit (all combinations, "fit4c_all").

    3. Decide if this fit is better than the previous.

    After loop over combintations: Write results of the Kalman kitematic fit of the best combination ("fit4c_best" branches).

    If there is a fit result, write the MC truth topology for this event. Also increment fCutFlow_NFitOK counter if fit worked.

(Inherited) initialize step of Algorithm.

This function is called only once, when the algorithm is initialised.

Remarks
Define and load NTuples here. Other NTuples have already been defined in the TrackSelector::initilize step prior to this this method.

  1. "mult_select": Multiplicities of selected particles

    1. "NKaonPos": Number of .

    2. "NKaonNeg": Number of .

    3. "NPionPos": Number of .

  2. "dedx_K" and "dedx_pi": energy loss PID branch. See TrackSelector::AddNTupleItems_Dedx for more info.

  3. "fit4c_*": results of the Kalman kinematic fit results. See TrackSelector::AddNTupleItems_Fit for more info.

  4. "topology": Add extra mass branches to the MC truth branch for the topoana package
