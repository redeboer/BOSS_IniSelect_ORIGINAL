Detailed Description

Base algorithm that creates a selection of charged and/or neutral tracks.

Your analysis has to be defined in derived algorithms, making use of the vectors that are filled in this algorithm. See the tutorial pages for more information.

The design of using maps of NTuple::Itemss is not ideal, as it does not (yet) allow enough flexibility.

◆ AddAndWriteCuts()

void TrackSelector::AddAndWriteCuts ( )
private

Write all cuts to a branch called "_cutvalues".

There will only be one entry per output file: a C-style array of size 3. Entry 0 of that array is the min value, entry 1 is the max value, and entry 2 is the recorded. As opposed to earlier versions of the TrackSelector (where the "_cutvalues" TTree contained 3 entries), we now use a C-style array (written through NTuple::addIndexedItem), because it allows to add up the number of counts when you have split up your jobs, which makes your output contain more than ROOT file. If you load these files as a TChain, you can get the total count of the CutObject by adding up all '2nd entries' (the counts) of each branch.

See also
BOSSOutputLoader::PrintCuts

This function encapsulates the addItem procedure for the energy loss branch ("dedx").

This method allows you to perform the same booking method for different types of charged particles (for instance 'all charged particles', kaons, and pions).

  • "Ngoodhits": Number of good hits (excluding overflow).
  • "Ntotalhits": Number of good hits (including overflow).
  • "chie": in case of electron ("number of \f$\sigma\f$ from \f$e^\pm\f$").
  • "chik": in case of kaon ("number of \f$\sigma\f$ from \f$K^\pm\f$").
  • "chimu": in case of muon ("number of \f$\sigma\f$ from \f$\mu^\pm\f$").
  • "chip": in case of proton ("number of \f$\sigma\f$ from \f$p^\pm\f$").
  • "chipi": in case of pion ("number of \f$\sigma\f$ from \f$\pi^\pm\f$").
  • "normPH": Normalized pulse height.
  • "p": Momentum of the track as reconstructed by MDC.
  • "probPH": Most probable pulse height from truncated mean.

Create a preselection of charged tracks (without cuts).

This method is used in TrackSelector::execute only. See fGoodChargedTracks for more information.

  1. Abort if fCreateChargedCollection has been set to false. This is decided in the derived class.

  2. Clear fGoodChargedTracks collection vector and initialize PID instance.

  3. @ Abort if there are no charged tracks in the fEvtRecEvent track collection.

  4. Loop over the charged tracks in the loaded fEvtRecEvent track collection. The first part of the set of reconstructed tracks are the charged tracks.

    1. Get MDC information

      Todo:
      Check if helixip.pivot(fVertexPoint) doesn't affect fVertexPoint.

    2. Apply charged track cuts:

      • coordinate of the collision point has to be within cut_Vz0_max

      • radius in plane has to be less than cut_Vr0_max

      • coordinate of the distance to the interaction point has to be within cut_Rvz0_max

      • distance to the interaction point in plane has to be within cut_Rvxy0_max

      • distance to the interaction point in plane has to be within cut_costheta_max

    3. Write charged track vertex position info ("charged" branch)

    4. Write dE/dx PID information ("dedx" branch)

    5. Write Time-of-Flight PID information ("tof*" branch)

Create a preselection of Monte Carlo truth tracks.

This method is used in TrackSelector::execute only. It is used to fill the fMcParticles vector with a selection of McParticle pointers. This collection starts with the initial cluster (e.g. ) and continues with the rest of the decay chain. Only then is it possible to use CreateMCTruthSelection, so it is called at the end.

See also
fMcParticles
WriteMCTruthForTopoAna
CreateMCTruthSelection
Returns
Returns true if a collection of Event::McParticles was successfully created.

  1. Abort if input file is not MC generated (that is, if the run number is not negative).

  2. Abort if "write_topology" job switch has been set to false.

  3. Clear fMcParticles vector.

  4. Load McParticelCol from "/Event/MC/McParticleCol" directory in "FILE1" input file and abort if does not exist.

  5. Loop over collection of MC particles (Event::McParticleCol). For more info on the data available in McParticle, see here. Only add to fMcParticles if the McParticle satisfies:

    • Skip if the track is not a primary particle (has no mother). The initial meson to which the beam is tuned is included, because its mother is a cluster or string.

    • Skip if the track is not from the generator. This means that it is simulated in the detectors, but did not come from the event generator.

    • Only start recording after we have passed the initial simulation cluster (code 91) or string (code 92). The next particle after this cluster or string will be the meson to which the beam is tuned (e.g. ).
      See also
      NTupleTopoAna::IsInitialCluster.

    • Add the pointer to the fMcParticles collection vector for use in the derived algorithms.

  6. (For the derived class:)
    Create selections of specific MC truth particles using CreateMCTruthSelection. Will not be performed if not specified in the derived algorithm.

◆ execute()

StatusCode TrackSelector::execute ( )

This method is called for each event.

Load DST input file

  1. Load headers from the input file.

    • Namespace EventModel
    • Class EvtRecEvent
    • Type definition EvtRecTrackCol
    • Type definition Event::McParticleCol

  2. Log run number, event number, and number of events.

  3. Increase counter objects (see CutObject).

  4. Set vertex origin.

Create track collections

  1. Perform CreateChargedCollection.
  2. Perform CreateNeutralCollection.
  3. Remarks
    You should call CreateMCtruthCollection in the derived class.

    Write event info

  1. Write general multiplicities ("mult" branch)
  2. Write information about the interaction point ("vertex" branch)

◆ initialize()

StatusCode TrackSelector::initialize ( )

(Inherited) initialize step of Algorithm.

This function is called once in the beginning of each run. Define and load NTuples here. The NTuples will become the TTrees in the eventual ROOT file, the added NTuple::Items will be the branches of those trees.

  1. "mult": Multiplicities of the total event

    1. "Ntotal": Total number of events per track.

    2. "Ncharge": Number of charged tracks.

    3. "Nneutral": Number of charged tracks.

    4. "NgoodNeutral": Number of 'good' neutral tracks (if performed).

    5. "NgoodCharged": Number of 'good' charged tracks (if performed).

  2. "vertex": Vertex info

    1. "vx0": coordinate of the collision point.

    2. "vy0": coordinate of the collision point.

    3. "vz0": coordinate of the collision point.

  3. "charged": Charged track info.

    1. "vx": coordinate of the secondary vertex as determined by MDC.

    2. "vy": coordinate of the secondary vertex as determined by MDC.

    3. "vz": coordinate of the secondary vertex as determined by MDC.

    4. "vr": Distance from origin in plane.

    5. "rvxy": Nearest distance to interaction point in plane.

    6. "rvz": Nearest distance to interaction point in direction.

    7. "rvphi": Angle in the plane.

      Todo:
      Get explanation of geometry (angle) definitions in an event.

    8. "phi": Helix angle of the particle.

    9. "p": Momentum of the track.

  4. "neutral": Neutral track info.

    1. "E": Energy of the neutral track as determined by the EM calorimeter.

    2. "x": -coordinate of the neutral track according to the EMC.

    3. "y": -coordinate of the neutral track according to the EMC.

    4. "z": -coordinate of the neutral track according to the EMC.

    5. "phi": -angle of the neutral track according to the EMC.

    6. "theta": -angle of the neutral track according to the EMC.

    7. "time": Time of the neutral track according to the EMC.

      Todo:
      Investigate what this parameter precisely means.

  5. "dedx": energy loss PID branch. See TrackSelector::AddNTupleItems_Dedx for more info.

  6. "ToFEC", "ToFIB", and "ToFOB": information from the three Time-of-Flight detectors. See TrackSelector::AddNTupleItems_Tof for more info.

  7. "topology": Monte Carlo truth for TopoAna package.

  8. "PID": Track PID information.

    1. "p": Momentum of the track as reconstructed by MDC.

    2. "cost": Theta angle of the track.

    3. "chiToFIB": of the inner barrel ToF of the track.

    4. "chiToFEC": of the end cap ToF of the track.

    5. "chiToFOB": of the outer barrel ToF of the track.

    6. "chidEdx": of the energy loss of the identified track.

    7. "prob_K": Probability that the track is from a kaon according to the probability method.

    8. "prob_e": Probability that the track is from a electron according to the probability method.

    9. "prob_mu": Probability that the track is from a muon according to the probability method.

    10. "prob_p": Probability that the track is from a proton according to the probability method.

    11. "prob_pi": Probability that the track is from a pion according to the probability method.
