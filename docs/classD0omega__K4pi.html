NTuples (eventual TTrees and their branches). NTuples are like vectors, but its members do not necessarily have to be of the same type. In this package, the NTuples are used to store event-by-event information. Its values are then written to the output ROOT file, creating a ROOT TTree. In that sense, each NTuple here represents one TTree within that output ROOT file, and each NTuple::Item represents its leaves. The name of the leaves is determined when calling NTuple::addItem. Note that the NTuple::Items are added to the NTuple during the D0omega_K4pi::initialize() step. This is also the place where you name these variables, so make sure that the structure here is reflected there!

StatusCode D0omega_K4pi::execute ( )

Inherited execute method of the Algorithm. This function is called for each event.

  1. Create log stream (MsgStream class)

  2. Load next event from DST file

  3. Set vertex origin

  4. LOOP OVER CHARGED TRACKS: select charged tracks (eventual pions)

    Apply cut: cut number of charged tracks and net charge

  5. LOOP OVER NEUTRAL TRACKS: select photons ** Uses fNCut3 counter**: number of good photons has to be 2 at least. The second part of the set of reconstructed events consists of the neutral tracks, that is, the photons detected by the EMC (by clustering EMC crystal energies). Each neutral track is paired with each charged track and if their angle is smaller than a certain value (here, 200), the photon track is stored as 'good photon' (added to iGam).

  6. Check charged track dEdx PID information

  7. Check charged track ToF PID information

  8. Get 4-momentum for each photon

  9. Get 4-momentum for each charged track

    Apply cut: PID

  10. Loop over each gamma pair and store total energy

  11. Get vertex fit

  12. Apply Kalman 4-constrain kinematic fit

    Apply cut: fit4c passed and ChiSq less than fMaxChiSq.

  13. Apply Kalman kinematic fit

    Apply cut: fit5c passed and ChiSq less than fMaxChiSq.

  14. Get MC truth
