Detailed Description

WIP Analyse events.

Based on the original RhopiAlg package, but now making use of the TrackSelector base algorithm and its NTuple booking procedure.

Specification of TrackSelector::CreateMCTruthSelection.

Create selection of MC truth particles by looping over the collection of MC particles created by TrackSelector::CreateMCTruthCollection().

  1. Abort if input file is not from a Monte Carlo simulation (that is, if the run number is not negative).
  2. Abort if "write_fit_mc", has been set to false.
  3. Clear MC truth particle selections.
  4. Loop over fMcParticles collection of MC truth particles and fill the selections.

◆ execute_rest()

StatusCode rhopi_pipigg::execute_rest ( )
virtual

Inherited execute method of the Algorithm. This function is called for each event.

  1. Charged track cut: Apply a strict cut on the number of particles. Only 2 charged tracks in total.

  2. Net charge cut: Apply a strict cut on the total charge detected in the detectors. If this charge is not , this means some charged tracks have not been detected.

  3. Create selection of charged tracks.

    1. Initialise PID and skip if it fails:

      • use probability method

      • use and the three ToF detectors. Since BOSS 7.0.4, ParticleID::useTofCorr should be used for ToF instead of e.g. useTof1.

      • identify only pions

      • use

    2. Write Particle Identification information of all tracks

    3. Identify type of charged particle and add to related vector: this package: only pions.

      A cut is then applied on whether the probability to be a pion (or kaon) is at least fCut_PIDProb_min (see eventual settings in rhopi_pipigg.txt).

      Finally, the particle ID of the RecMdcKalTrack object is set to pion

      and the pions ( ) are added to the respective vector of pions.

  4. Create selection neutral tracks (photons)

    1. Get EM calorimeter info.

    2. Find angle differences with nearest charged pion.

      • Get the extension object from MDC to EMC.
      • Get angles in radians.

    3. Convert angles from radians to degrees.

    4. Write photon info ("photon" branch).

    5. Apply angle cut on the photons: you do not want to photons to be too close to any charged track to avoid noise from EMC showers that came from a charged track.

    6. Add photon track to vector for gammas

  5. PID cut: apply a strict cut on the number of the selected particles. Only continue if:

    1. at least 2 photons ( 's)

    2. 1 negative pion ( )

    3. 1 positive pion ( )

  6. Create selection of MC truth particles by looping over the collection of MC particles created in TrackSelector::execute(). See here for a list of PDG codes.

  7. Write the multiplicities of the selected particles.

  8. Write PID information ("dedx_pi" branch)

  9. Loop over MC truth of final decay products.

    1. Only continue if the two photons are different.

    2. Check topology: only consider that combination which comes from .

    3. Write 'fake' fit results, that is, momenta of the particles reconstructed from MC truth.

  10. Perform Kalman 4-constraint Kalman kinematic fit for all combinations

    1. Reset best fit parameters (see KKFitResult_rhopi_pipigg).

    2. Loop over all combinations of , , , and .

      1. Only continue if we are not dealing with the same kaon.

      2. Get Kalman tracks.

      3. Get W-tracks.

      4. Initiate vertex fit.

      5. Test vertex fit.

      6. Get Kalman kinematic fit for this combination and store if better than previous one.

        1. Apply max. cut (determined by fCut_PIDChiSq_max).

        2. Construct fit result object for this combintation.

        3. Write results of the Kalman kinematic fit.

        4. Write MC truth collection for the topoana package.

  11. Perform Kalman 5-constraint Kalman kinematic fit for all combinations.

    1. Reset best fit parameters.

    2. Loop over all combinations of pi-, pi+, and gamma.

      1. Only continue if we are not dealing with the same kaon.

      2. Get Kalman tracks.

      3. Get W-tracks.

      4. Initiate vertex fit.

      5. Test vertex fit.

      6. Get Kalman kinematic fit for this combination and store if better than previous one.

        Remarks
        5th constraint: resonance

        1. Apply max. cut (determined by fCut_PIDChiSq_max).

        2. Construct fit result object for this combintation.

        3. Write results of the Kalman kinematic fit.

◆ initialize_rest()

StatusCode rhopi_pipigg::initialize_rest ( )
virtual

(Inherited) initialize step of Algorithm.

This function is called only once, when the algorithm is initialised.

Remarks
Define and load NTuples here. Other NTuples have already been defined in the TrackSelector::initilize step prior to this this method.

  1. "mult_select": Multiplicities of selected particles

    1. "NPhotons": Number of 's.

    2. "NPionNeg": Number of .

    3. "NPionPos": Number of .

  2. "dedx_pi": energy loss PID branch. See TrackSelector::AddNTupleItems_dedx for more info.

  3. "fit4c" and "fit5c": results of the Kalman kinematic fit results. See TrackSelector::AddNTupleItems_Fit for more info.

  4. "photon": information of the selected photons

    1. "E": Energy of the photon.

    2. "px": component of the 4-momentum of the photon (computed from the detected angles).

    3. "py": component of the 4-momentum of the photon (computed from the detected angles).

    4. "pz": component of the 4-momentum of the photon (computed from the detected angles).

    5. "phi": Smallest angle between the photon and the nearest charged pion.

    6. "theta": Smallest angle between the photon and the nearest charged pion.

    7. "angle": Smallest angle between the photon and the nearest charged pion.
