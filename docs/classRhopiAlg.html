NTuples (eventual TTrees and their branches). NTuples are like vectors, but its members

do not necessarily have to be of the same type.

In this package, the NTuples are used to store event-by-event information. Its values are then written to the output ROOT file, creating a ROOT TTree. In that sense, each NTuple here represents one TTree within that output ROOT file, and each NTuple::Item represents its leaves. The name of the leaves is determined when calling NTuple::addItem. Note that the NTuple::Items are added to the NTuple during the RhopiAlg::initialize() step. This is also the place where you name these variables, so make sure that the structure here is reflected there!

Detailed Description

Commented version of the RhopiAlg example BOSS package (version 00-00-23).

The RhopiAlg example package particularly teaches the use of the Kalman kinematic fit procedure.

Remarks
In addition to the more extensive comments, some additional parameters, such as ToF without particle hypothesis, have been added.

◆ execute()

StatusCode RhopiAlg::execute ( )

Inherited execute method of the Algorithm. This function is called for each event.

  1. Create log stream (MsgStream class)

  2. Load next event from DST file ** Uses Ncut0 counter**: no cut applied yet.

  3. Set vertex origin

  4. LOOP OVER CHARGED TRACKS: select charged tracks (eventual pions) ** Uses Ncut1 counter**: nGood has to be 2, mdcTrk->charge() has to be more than 0.

  5. LOOP OVER NEUTRAL TRACKS: select photons ** Uses Ncut2 counter**: number of good photons has to be 2 at least.

  6. Check charged track dEdx PID information

  7. Check charged track ToF PID information

  8. Get 4-momentum for each photon

  9. Get 4-momentum for each charged track

  10. RhopiAlg without PID (optional: needs to be uncommented)

    ** Uses Ncut3 counter**: ipip.size() * ipim.size() cannot be 1.

  11. Loop over each gamma pair and store total energy

  12. Default is pion, for other particles (optional: needs to be uncommented)

  13. Test vertex fit

  14. Apply Kalman 4-constrain kinematic fit

    ** Uses Ncut4 counter**: fit4c passed and ChiSq less than fMaxChiSq.

  15. Apply Kalman kinematic fit ** Uses Ncut5 counter**: Kalman kinematic fit 5c is successful. ** Uses Ncut6 counter**: (cut on invariant mass window).
