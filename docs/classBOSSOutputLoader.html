Detailed Description

General container for BOSS analysis output.

This object only needs a path of a directory containin ROOT files from the initial analysis or of one such root file. All TTrees and the branches in these files will be loaded automatically, so you don't have to worry about names of those trees. This object also comes with several methods that faciliate e.g. plotting of the TTree contents.

Motivation: To investigate the contents of the ROOT file, you first need to know which TTrees and branches it contains. If you simply construct the BOSSOutputLoader by giving it a file name, all TTrees will be loaded automatically as well as addresses for each of their branches. Five the constructer true as its second argument, and the names of these TTrees, their branches, and the types of these branches (behind the slash / after the name) will be printed to the terminal. Do this if your macro throws an exception, because this probably means that you use the wrong names for the trees and or the branches further on in the macro.

Constructor that opens a TFile and unordered_maps its contents.

  1. Make an inventory of ROOT files in input directory and add filenames to the fFileNames list.
  2. If path ends in ".root", treat as single ROOT file and attempt to load it.

  • Abort if path cannot be loaded as ROOT file.

  • Extract directory and file name from input path.

  • Add resulting file name to fFileNames and continue loading as TChain.

  1. Else, treat path as path to directory and make an inventory of ROOT files in this directory.

  • Remove trailing slash / from fDirectoryPath

  • Open file directory using TSystemDirectory.

  • Abort if directory is empty.

  • Loop over list of files in directory

    and add file name if file exists and is no zombie.

  1. Execute LoadChains now that the fFileNames list has been loaded.

void BOSSOutputLoader::LoadChains ( bool  print_branches = true,
bool  print_averages = false 
)

Create a list of TChains based on the files in the directory you are loading.

If the ROOT TFile could be successfully loaded, this method will generate a std::unordered_map of pointers to all TChains in the main folder of the ROOT file. The first file in the directory is used to make an inventory of all TTrees within the file.

  1. Check whether there are files in the fFileNames list.
  2. Load first ROOT file fFileNames list.
  3. Go through keys in the file and add create ChainLoader if the key is a TTree.

  • Attempt to cast loaded TObject to TKey.

  • Read TKey and attempt to cast to TTree.

  • Create ChainLoader object (with underlying TChain) from the loaded TTree.

  • Loop over file names and at them to the ChainLoader object.

  • Book branches using ChainLoader::BookAddresses.

  1. Print terminal output

◆ PrintCuts()

void BOSSOutputLoader::PrintCuts ( )

Function that allows you to print the cut definitions if they have been stored in the output ROOT file.

  1. Abort if there is no TChain "_cutvalues" or if it is empty.

  2. Create an inventory of branches in the form of a std::map. The keys are the branch names, the values are double vectors of size 3 that will contain the loaded values. We use a sorted map so that the keys are sorted automatically. At the same time, the maximum number of characters in all of the cut names is also determined.

  3. CASE1. If "_cutvalues" contains a branch called "index", assume that the TTree contains double arrays.

    1. SetBranchAddress for the array of this branch. Abort if SetBranchAddress does not return 0.

    2. Entry 0: this is information written from the CutObject to the first file of the TChain loaded here. We use it to load the min and max value (array entries 0 and 1).
      See also
      TrackSelector::AddAndWriteCuts

    3. Get the total count value by adding up each 3rd entry of the array.

    4. Drop address allocated for this branch.

  4. If TChain "_cutvalues" does not contain a branch called "index", it is probably created by an older version of the TrackSelector and we need to use one of the following methods.

    1. Abort if "_cutvalues" contains 2 entry. This means something went wrong while writing this NTuple.

    2. CASE2. If "_cutvalues" contains only 1 entry, only print a list of cut paramters and their values:

      • For each branch, add up the entries

      • Entry 0 of the "_cutvalues" tree is considered to be that value.

    3. CASE3. If "_cutvalues" contains at least 3 entries, consider entry 0 to be the min value, entry 1 to be the max, and entry 2 to be count (the number of events or tracks that passed the cut).

      1. Get values from entry 0 (min).

      2. Get values from entry 1 (max).

      3. Get values from entry 2 (count). In case of an hadded file, add up the values of entry 2, 5, 8, etc.

  5. Print loaded values as a table: one row per parameters.

    1. Get width of columns (w_name has to be redone because of the table headers).

    2. Print table header with four columns.

    3. Print horizontal line beneath it.

    4. Create a list of cut names ordered by counter (using std::list::sort and a lambda).

    5. Print the table:

      1. Print a horizontal line just before counter "N_events".

      2. Column 1: cut name.

      3. Column 2: minimum, if available.

      4. Column 3: maximum, if available.

      5. Column 4: counter, if available.

  6. Reset branches to avoid a crash at the end of your executable.
